⏺ Here's a practical end-to-end test plan, organized by flow. Work through these
   in order — each section builds on the previous.

  ---
  Setup

  1. Run the dev server: npm run dev
  2. Have two browser windows ready — one for a system_admin user, one for a
  standard_user
  3. Keep the browser console open throughout (F12) — watch for errors

  ---
  1. Auth & Routing

  Desktop (> 768px):
  - Log in → should land on /dashboard
  - Log out → should return to /
  - Visit a protected route while logged out → should redirect to /

  Mobile (resize to < 768px or DevTools mobile):
  - Log in → should land on /mobile/dashboard
  - Bottom nav should be visible (Home, Payments, Renters, Leases, Issues)
  - Tapping each bottom nav item navigates correctly
  - Sidebar should NOT be visible on mobile (it's the hamburger menu instead)

  Role gates:
  - Log in as standard_user → /settings and /reports should redirect away
  - Log in as system_admin → sees System Admin dashboard instead of regular
  Dashboard

  ---
  2. Dashboard

  - Stats cards show real numbers (not zeros, not obviously fake data like
  "45,230")
  - Occupancy Rate shows "X / Y rooms" subtitle
  - Overdue Payments turns red if > 0
  - Recent Payments list shows real payments (not mock names like "Bongani
  Mthembu")
  - Portfolio Summary row: Facilities / Total Rooms / Available Rooms /
  Collection Rate
  - Refresh button spins and reloads data
  - Quick Action buttons navigate to the right pages

  ---
  3. Global Search (Ctrl+K)

  - Press Ctrl+K (or Cmd+K on Mac) → search modal opens
  - Type a renter's name → results appear with type badges (Renter / Room /
  Lease)
  - Click a result → navigates to that page and modal closes
  - Press Escape → modal closes
  - Click backdrop → modal closes
  - Search icon in Header also opens the modal

  ---
  4. Facilities

  - Create a facility (name, address, type)
  - It appears in the list

  ---
  5. Rooms

  - Create a room (attach to a facility, set rent, status = available)
  - Room appears in list with correct status badge
  - Click the delete button → confirmation dialog appears (not window.confirm)
  - Cancel → room remains; Confirm → room is deleted

  ---
  6. Renters

  - Create a renter with a phone number starting with 0 (e.g. 0821234567)
  - View the renter → a WhatsApp icon/link should appear next to the phone
  number
  - Click it → opens https://wa.me/27821234567 in a new tab
  - Same link should appear in both card view and table view

  ---
  7. Leases

  Create:
  - Create a lease linking a renter to a room
  - Lease appears in list as Active

  Rent Increase:
  - Find an active lease → click "Update Rent" (TrendingUp icon)
  - Modal shows current rent vs. new rent with live % calculation
  - Enter > 15% increase → warning banner appears
  - Submit → confirm the terms.monthlyRent updated in Firestore

  Termination:
  - Find an active lease → click Terminate
  - Termination form shows renter name, room number, facility name (not
  blank/undefined)
  - Submit termination → lease status updates

  ---
  8. Payments

  Late fee preview:
  - Open PaymentCapture for an overdue payment
  - An orange banner should show: "X days late · Late fee: R Y · Total due: R Z"
   before you submit

  Duplicate guard:
  - Try to record a second payment for the same renter in the same month
  - A yellow inline warning should appear (not a browser alert())
  - Submit is blocked until you tick "I understand" checkbox
  - Tick checkbox → submit becomes available

  Receipt modal:
  - Submit a valid payment → a receipt modal appears with receipt number,
  amount, method, date
  - Print button triggers window.print()
  - Close → navigates away normally

  capturedBy field:
  - After submitting a payment, check the Firestore document for that payment
  month
  - capturedBy should be your actual Firebase UID (not the string
  "current_user_id")

  ---
  9. Complaints (Maintenance)

  - Create a complaint with Category = Maintenance
  - A sub-category dropdown appears (Plumbing, Electrical, Staff, Paint,
  Windows/Doors, Other)
  - Select a sub-category, attach a photo (Upload or camera)
  - Submit → complaint appears in list with sub-category shown
  - Open the complaint detail → photo renders, sub-category badge visible

  ---
  10. Reports

  - Navigate to /reports
  - Two charts should render: "12-Month Revenue Overview" (bar) and "Facility
  Performance" (horizontal bar)
  - Click the PDF button → print dialog opens, charts are visible in print
  preview
  - Sidebar and nav should be hidden in print preview (check @media print CSS)

  ---
  11. Notifications

  - Navigate to /notifications
  - Click the ✓ (mark as read) on a notification → no browser alert(), it just
  marks as read silently
  - Click the trash icon → no browser alert(), notification is deleted
  - Click "Mark all as read" → same, no alert()
  - If any of the above fail → a toast should appear bottom-right (not an alert)

  ---
  12. Toast Notifications

  While testing the above, confirm:
  - Success actions (payment captured, lease created, etc.) show a green toast
  bottom-right
  - Error conditions show a red toast
  - Toasts auto-dismiss after ~4 seconds
  - Toasts can be manually dismissed by clicking ✕

  ---
  13. Sidebar Badges

  - Sidebar shows a red badge count on:
    - Payments (overdue count)
    - Payment Approvals (pending approvals)
    - Notifications (unread count)
    - Complaints (open complaints)
  - Badges update when you navigate between pages
  - On mobile, sidebar closes automatically when you tap a nav item

  ---
  Common Things to Watch For
